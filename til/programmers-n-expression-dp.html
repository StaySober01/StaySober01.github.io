프로그래머스 N으로 표현: DP로 가능한 값 쌓기

TIL · 프로그래머스 문제 풀이 · 2026.07.10

프로그래머스의 N으로 표현 문제는 숫자 N과 사칙연산만으로 목표 숫자를 만드는 최소 사용 횟수를 구하는 문제입니다. 핵심은 N을 몇 번 사용했는지에 따라 만들 수 있는 모든 값을 집합으로 저장하고, 작은 사용 횟수의 결과를 조합해 더 큰 사용 횟수의 결과를 만드는 것입니다.

문제 이해

예를 들어 N = 5, number = 12라면 다음처럼 5와 사칙연산만 사용해 12를 만들 수 있습니다. 

12 = 5 + 5 + (5 / 5) + (5 / 5)
12 = 55 / 5 + 5 / 5
12 = (55 + 5) / 5

각각 5를 6번, 5번, 4번 사용합니다. 이 중 최소 사용 횟수는 4이므로 답은 4입니다. 문제는 이 최소 사용 횟수를 구하되, 8번을 넘겨야 만들 수 있다면 -1을 반환하라고 합니다.

제한이 "최솟값이 8보다 크면 -1"로 작게 잡혀 있기 때문에, 1번부터 8번까지 만들 수 있는 값을 전부 모아보는 방식이 가능합니다.

DP 상태 정의

이 문제에서 가장 중요한 문장은 다음입니다. 

dp[i] = 숫자 N을 i번 사용해서 만들 수 있는 모든 숫자의 집합

vector<set<int>> dp(9)를 사용하면 dp[1]부터 dp[8]까지 관리할 수 있습니다. set을 쓰는 이유는 같은 값이 여러 식에서 반복해서 만들어져도 한 번만 저장하기 위해서입니다.

N = 5일 때 예시는 이렇게 볼 수 있습니다. 

dp[1] = { 5 }

dp[2] = {
    55,
    5 + 5,
    5 - 5,
    5 * 5,
    5 / 5
}

dp[3], dp[4]도 같은 방식입니다. 이미 만들어둔 작은 집합들을 조합하면 더 큰 사용 횟수의 집합을 만들 수 있습니다.

반복 숫자 넣기

사칙연산 조합만 생각하면 55, 555 같은 숫자를 놓치기 쉽습니다. 그래서 각 i마다 N을 i번 이어 붙인 값을 먼저 넣어야 합니다. 

int repeatNum = 0;

for (int j = 0; j < i; j++)
{
    repeatNum = repeatNum * 10 + N;
}

dp[i].insert(repeatNum);

N = 5라면 i = 1일 때 5, i = 2일 때 55, i = 3일 때 555가 만들어집니다.

작은 집합 조합하기

dp[i]는 i번을 둘로 나눈 조합에서 만들 수 있습니다. 예를 들어 dp[4]는 dp[1] + dp[3], dp[2] + dp[2], dp[3] + dp[1] 조합에서 나옵니다. 

for (int j = 1; j < i; j++)
{
    int leftCount = j;
    int rightCount = i - j;

    for (int a : dp[leftCount])
    {
        for (int b : dp[rightCount])
        {
            dp[i].insert(a + b);
            dp[i].insert(a - b);
            dp[i].insert(a * b);

            if (b != 0)
                dp[i].insert(a / b);
        }
    }
}

여기서 a - b와 a / b는 순서가 중요합니다. 그래서 j를 1부터 i - 1까지 모두 돌며 dp[1] + dp[3]과 dp[3] + dp[1]을 각각 확인합니다. 나눗셈은 0으로 나눌 수 없으므로 b != 0 조건을 반드시 둡니다.

최종 코드

각 사용 횟수 i에 대해 가능한 값을 모두 만든 뒤, 그 집합 안에 number가 있으면 바로 i를 반환합니다. 작은 i부터 확인하기 때문에 처음 발견한 순간이 최소 사용 횟수입니다. 

#include <string>
#include <vector>
#include <set>

using namespace std;

int solution(int N, int number) {
    vector<set<int>> dp(9);

    for (int i = 1; i <= 8; i++) {
        int repeatNum = 0;

        for (int j = 0; j < i; j++)
        {
            repeatNum = repeatNum * 10 + N;
        }

        dp[i].insert(repeatNum);

        for (int j = 1; j < i; j++) {
            int leftCount = j;
            int rightCount = i - j;

            for (int a : dp[leftCount])
            {
                for (int b : dp[rightCount])
                {
                    dp[i].insert(a + b);
                    dp[i].insert(a - b);
                    dp[i].insert(a * b);

                    if (b != 0)
                        dp[i].insert(a / b);
                }
            }
        }

        if (dp[i].count(number)) return i;
    }

    return -1;
}

풀이 체크포인트

  • dp[i]의 의미를 먼저 고정한다

    i번째 값이 아니라, N을 i번 사용해서 만들 수 있는 모든 값의 집합입니다.

  • 이어 붙인 숫자를 별도로 넣는다

    55, 555는 사칙연산 조합에서 자동으로 나오지 않으므로 각 단계에서 직접 추가합니다.

  • 뺄셈과 나눗셈은 순서를 놓치지 않는다

    a - b와 b - a는 다르고, a / b와 b / a도 다릅니다. 모든 분할을 순서대로 확인해야 합니다.

  • 최소 횟수는 조기 반환으로 처리한다

    i = 1부터 증가시키며 검사하므로, 목표값을 처음 찾은 순간의 i가 곧 정답입니다.

정리

  • dp[i]는 N을 i번 사용해서 만들 수 있는 값들의 집합이다.
  • 각 단계마다 N, NN, NNN처럼 이어 붙인 값을 먼저 넣는다.
  • dp[j]와 dp[i - j]의 모든 값을 사칙연산으로 조합해 dp[i]를 채운다.
  • 나눗셈은 0으로 나누지 않도록 조건을 둔다.
  • 1번부터 8번까지 확인하면서 목표값을 처음 찾으면 그 횟수를 반환하고, 끝까지 없으면 -1을 반환한다.

이 문제는 DP를 "최적값 하나를 저장하는 배열"로만 보지 않고, "현재 사용 횟수로 만들 수 있는 상태 전체를 저장하는 집합"으로 확장해서 생각하는 연습이 됩니다. 가능한 값을 차곡차곡 쌓아두면, 완전탐색처럼 보이는 문제도 제한 안에서 깔끔하게 정리할 수 있습니다.
