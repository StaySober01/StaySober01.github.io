DFS와 BFS 비교

그래프 탐색은 시작 정점에서 간선을 따라 도달 가능한 정점을 방문하는 과정입니다. DFS와 BFS는 같은 정점을 탐색하더라도 다음 방문 대상을 선택하는 기준과 필요한 자료구조가 다릅니다.

구분DFSBFS
탐색 방식한 경로를 끝까지 탐색가까운 정점부터 층별 탐색
자료구조재귀 호출 또는 스택큐
최단 거리일반적으로 보장하지 않음가중치 없는 그래프에서 보장
주요 용도백트래킹, 사이클, 연결 요소최소 이동 횟수, 전파, 레벨 탐색
시간복잡도O(V + E)O(V + E)

DFS는 깊게 파고들고, BFS는 시작점에서 가까운 순서로 퍼져 나갑니다. 이 차이가 문제에 맞는 탐색 방법을 결정합니다.

그래프 표현과 방문 순서

다음 트리 모양의 무방향 그래프를 왼쪽 정점부터 방문한다고 가정합니다.

        1
      /   \
     2     3
    / \   / \
   4   5 6   7
  • DFS: 1 → 2 → 4 → 5 → 3 → 6 → 7
  • BFS: 1 → 2 → 3 → 4 → 5 → 6 → 7

이 순서는 유일한 답이 아닙니다. 인접 정점을 저장한 순서가 다르면 DFS와 BFS의 세부 방문 순서도 달라집니다. 문제에서 작은 번호부터 방문하라고 요구하면 인접 리스트를 정렬하거나, 스택에 넣는 순서를 조정해야 합니다.

DFS: 깊이 우선 탐색

DFS(Depth-First Search)는 한 방향을 선택해 더 이상 갈 수 없을 때까지 내려갑니다. 막히면 이전 분기점으로 되돌아가 아직 방문하지 않은 다른 경로를 탐색합니다.

재귀 호출로 구현하기

#include <iostream>
#include <vector>

void Dfs(
    int current,
    const std::vector<std::vector<int>>& graph,
    std::vector<bool>& visited)
{
    visited[current] = true;
    std::cout << current << ' ';

    for (int next : graph[current]) {
        if (!visited[next]) {
            Dfs(next, graph, visited);
        }
    }
}

재귀 호출은 함수의 지역 변수와 복귀 위치를 호출 스택에 저장합니다. Dfs(4)가 끝나면 호출 스택에 남아 있는 Dfs(2)로 돌아가 다음 인접 정점을 확인하므로, 별도의 스택을 만들지 않아도 깊이 우선 순서가 만들어집니다.

스택으로 직접 구현하기

#include <stack>
#include <vector>

void Dfs(int start, const std::vector<std::vector<int>>& graph)
{
    std::vector<bool> visited(graph.size(), false);
    std::stack<int> stack;
    stack.push(start);

    while (!stack.empty()) {
        int current = stack.top();
        stack.pop();

        if (visited[current]) {
            continue;
        }

        visited[current] = true;
        std::cout << current << ' ';

        // 작은 번호를 먼저 꺼내려면 큰 번호부터 넣는다.
        for (int i = static_cast<int>(graph[current].size()) - 1;
             i >= 0;
             --i) {
            int next = graph[current][i];
            if (!visited[next]) {
                stack.push(next);
            }
        }
    }
}

스택은 나중에 넣은 값이 먼저 나오는 LIFO 구조이므로 원하는 방문 순서의 역순으로 인접 정점을 넣습니다. 재귀 구현은 간결하지만 그래프가 매우 깊으면 호출 스택이 넘칠 수 있습니다. 입력 크기가 크거나 깊이를 예측하기 어렵다면 명시적 스택이 안전합니다.

BFS: 너비 우선 탐색

BFS(Breadth-First Search)는 시작점과 직접 연결된 정점을 모두 방문한 뒤, 그다음 거리의 정점을 방문합니다. 먼저 넣은 값을 먼저 꺼내는 FIFO 큐가 이 순서를 만듭니다. 

#include <iostream>
#include <queue>
#include <vector>

void Bfs(int start, const std::vector<std::vector<int>>& graph)
{
    std::vector<bool> visited(graph.size(), false);
    std::queue<int> queue;

    visited[start] = true;
    queue.push(start);

    while (!queue.empty()) {
        int current = queue.front();
        queue.pop();
        std::cout << current << ' ';

        for (int next : graph[current]) {
            if (!visited[next]) {
                visited[next] = true;
                queue.push(next);
            }
        }
    }
}

BFS에서는 큐에 넣는 순간 방문 처리하는 것이 중요합니다. 큐에서 꺼낼 때까지 방문 표시를 미루면 여러 정점이 같은 정점을 발견해 큐에 중복 삽입할 수 있습니다. enqueue 시점에 표시하면 각 정점이 큐에 한 번만 들어갑니다. 

거리 0: 1
거리 1: 2, 3
거리 2: 4, 5, 6, 7

BFS와 최단 거리

BFS는 시작점에서 간선 수가 0개인 정점, 1개인 정점, 2개인 정점 순서로 탐색합니다. 따라서 모든 간선의 비용이 같은 그래프에서는 어떤 정점을 처음 발견했을 때의 거리가 최단 거리입니다. 

std::vector<int> BfsDistance(
    int start,
    const std::vector<std::vector<int>>& graph)
{
    std::vector<int> distance(graph.size(), -1);
    std::queue<int> queue;

    distance[start] = 0;
    queue.push(start);

    while (!queue.empty()) {
        int current = queue.front();
        queue.pop();

        for (int next : graph[current]) {
            if (distance[next] == -1) {
                distance[next] = distance[current] + 1;
                queue.push(next);
            }
        }
    }

    return distance;
}

-1은 아직 방문하지 않았음을 나타내며, 방문 배열과 거리 배열의 역할을 하나로 합칩니다.

  • 모든 간선 비용이 같음

    BFS를 사용합니다.

  • 간선 비용이 0 또는 1

    덱을 이용하는 0-1 BFS를 고려합니다.

  • 음수가 없는 서로 다른 비용

    다익스트라 알고리즘을 사용합니다.

  • 음수 간선이 존재할 수 있음

    조건에 따라 벨만–포드 같은 알고리즘을 검토합니다.

2차원 맵 탐색

코딩 테스트와 게임에서는 그래프가 인접 리스트가 아니라 격자로 주어지기도 합니다. 각 칸을 정점, 상하좌우 이동을 간선으로 생각하면 같은 DFS와 BFS를 적용할 수 있습니다. 

struct Position {
    int row;
    int column;
};

const int dr[4] = {-1, 1, 0, 0};
const int dc[4] = {0, 0, -1, 1};

std::vector<std::vector<int>> distance(
    rowCount,
    std::vector<int>(columnCount, -1));

std::queue<Position> queue;
distance[0][0] = 0;
queue.push({0, 0});

while (!queue.empty()) {
    Position current = queue.front();
    queue.pop();

    for (int direction = 0; direction < 4; ++direction) {
        int nextRow = current.row + dr[direction];
        int nextColumn = current.column + dc[direction];

        if (nextRow < 0 || nextRow >= rowCount ||
            nextColumn < 0 || nextColumn >= columnCount) {
            continue;
        }
        if (map[nextRow][nextColumn] == 0 ||
            distance[nextRow][nextColumn] != -1) {
            continue;
        }

        distance[nextRow][nextColumn] =
            distance[current.row][current.column] + 1;
        queue.push({nextRow, nextColumn});
    }
}

경계 확인, 이동 불가능한 칸 확인, 방문 여부 확인의 순서를 명확히 나누면 실수를 줄일 수 있습니다. 미로 최단 거리, 게임 맵 이동, 바이러스·불의 확산 시간, 특정 지점까지의 최소 이동 횟수가 대표적인 BFS 문제입니다. 연결된 영역이나 섬의 개수만 필요하다면 DFS와 BFS 모두 사용할 수 있습니다.

시간·공간 복잡도

인접 리스트로 표현한 그래프에서 DFS와 BFS는 각 정점을 최대 한 번 방문하고 각 간선을 확인하므로 시간복잡도는 O(V + E)입니다. 무방향 그래프에서는 하나의 간선이 양쪽 인접 리스트에 들어가 두 번 확인되지만 상수 배이므로 같은 복잡도로 표현합니다.

방문 배열은 O(V) 공간을 사용합니다. DFS는 최악의 경우 깊이만큼 호출 스택 또는 명시적 스택이 필요하고, BFS는 한 레벨의 많은 정점이 큐에 동시에 들어갈 수 있습니다. 두 방식 모두 최악의 추가 공간은 O(V)입니다. 인접 행렬을 사용하면 모든 정점 쌍을 확인할 수 있어 탐색 시간이 O(V²)가 될 수 있습니다.

선택 기준

BFS를 먼저 떠올릴 단서

  • 가중치 없는 그래프의 최단 거리 또는 최소 이동 횟수
  • 가장 빠른 시간, 몇 번 만에 도달하는지 묻는 문제
  • 시작점에서 가까운 순서나 단계별 확산이 중요한 문제

DFS를 먼저 떠올릴 단서

  • 가능한 경로와 모든 경우를 깊게 탐색하는 문제
  • 조합, 순열, 백트래킹과 상태 복원이 필요한 문제
  • 사이클 확인, 위상 정렬, 연결 요소 탐색

단순히 모든 정점을 방문하는 것이 목적이라면 둘 다 가능합니다. 이때는 그래프의 깊이, 필요한 방문 순서, 메모리 사용과 구현 편의성을 함께 보고 선택합니다.

자주 틀리는 부분

  • 방문 배열을 사용하지 않음

    사이클이 있는 그래프에서는 같은 정점을 계속 오가며 탐색이 끝나지 않을 수 있습니다.

  • BFS 방문 처리가 늦음

    큐에서 꺼낼 때가 아니라 큐에 넣을 때 방문 처리해 중복 삽입을 막습니다.

  • 정점 번호와 배열 크기가 맞지 않음

    정점 번호가 1부터 N이라면 인접 리스트와 방문 배열을 보통 N + 1 크기로 만듭니다.

  • 비연결 그래프를 한 번만 탐색함

    모든 정점을 순회하며 아직 방문하지 않은 정점에서 탐색을 다시 시작해야 합니다.

  • 방문 순서를 당연하게 가정함

    출력 순서 조건이 있으면 인접 리스트 정렬과 스택 삽입 순서를 확인합니다.

int componentCount = 0;

for (int node = 1; node <= vertexCount; ++node) {
    if (!visited[node]) {
        Dfs(node, graph, visited);
        ++componentCount;
    }
}

정리

  • DFS는 재귀 호출이나 스택을 사용해 한 경로를 끝까지 탐색한다.
  • BFS는 큐를 사용해 시작점에서 가까운 정점부터 탐색한다.
  • BFS는 모든 간선 비용이 같을 때 처음 발견한 거리를 최단 거리로 보장한다.
  • 2차원 맵도 각 칸을 정점, 이동을 간선으로 보면 같은 탐색을 적용할 수 있다.
  • 인접 리스트 기준 두 탐색의 시간복잡도는 O(V + E), 추가 공간은 최악의 경우 O(V)다.
  • 방문 처리 시점, 인덱스 범위, 연결 요소와 인접 정점 순서를 주의해야 한다.

DFS와 BFS를 자료구조 공식으로만 외우기보다 탐색의 진행 순서를 이해하면 문제의 표현에서 적절한 방법을 찾기 쉬워집니다. 깊은 경로와 상태 복원이 중요하면 DFS, 가까운 순서와 최소 이동이 중요하면 BFS가 자연스러운 출발점입니다.
