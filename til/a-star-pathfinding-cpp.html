A* 알고리즘: 휴리스틱 경로 탐색과 C++ 구현

TIL · 알고리즘과 자료구조 · 2026.07.18

다익스트라 알고리즘에 목적지까지의 예상 비용을 더하면 탐색을 목표 방향으로 유도할 수 있습니다. A*의 f(n) = g(n) + h(n)이 무엇을 의미하는지, 어떤 휴리스틱이 최단 경로를 보장하는지, 격자 맵에서 실제 경로를 어떻게 복원하는지 정리합니다.

A*의 핵심 아이디어

A*는 하나의 목적지를 향해 탐색할 때 다익스트라의 우선순위에 휴리스틱을 더한 최단 경로 알고리즘입니다. 각 정점 n의 우선순위는 다음 세 값으로 구성됩니다.

f(n) = g(n) + h(n)

g(n): 시작점에서 n까지 실제로 이동한 비용
h(n): n에서 목적지까지 남은 비용의 추정값
f(n): n을 거쳐 목적지에 도달할 것으로 예상한 총비용

우선순위 큐에서는 f가 가장 작은 정점을 먼저 꺼냅니다. 다익스트라가 시작점 주변을 비용 순서대로 넓게 확장한다면, A*는 h를 나침반처럼 사용해 목적지가 있을 법한 방향을 먼저 탐색합니다.

h(n)=0으로 두면 f(n)=g(n)이므로 A*는 다익스트라와 같아집니다. 즉 A*의 정확성과 성능 차이는 휴리스틱을 어떻게 설계하는지에서 생깁니다.

격자 예제로 탐색 과정 따라가기

S . . . .
. # # # .
. . . . G

S는 시작점, G는 목적지, #은 지나갈 수 없는 장애물입니다. 상하좌우 한 칸의 이동 비용이 1이라면 현재 좌표 (x, y)에서 목표 (gx, gy)까지의 맨해튼 거리를 사용할 수 있습니다.

h = |x - gx| + |y - gy|

현재 위치: (1, 1)
목적지:    (5, 3)
g = 3, h = |1 - 5| + |1 - 3| = 6
f = 3 + 6 = 9

시작점은 g(S)=0, f(S)=h(S)로 큐에 넣습니다. 이후 가장 작은 f를 가진 위치를 꺼내 이웃을 확인하고, 현재 위치를 거쳐 가는 newG가 더 작으면 g와 부모를 갱신합니다. 목적지가 큐에서 꺼내졌을 때 경로가 확정됩니다.

open에 시작점 추가
g[start] = 0

while open이 비어 있지 않음:
    f가 가장 작은 current를 꺼냄
    오래된 큐 항목이면 건너뜀

    if current == goal:
        parent를 역추적해 경로 반환

    for current와 연결된 next:
        newG = g[current] + cost(current, next)
        if newG < g[next]:
            parent[next] = current
            g[next] = newG
            open에 (newG + h(next), newG, next) 추가

이동 규칙에 맞는 휴리스틱 선택하기

휴리스틱공식적합한 이동 규칙
맨해튼 거리|dx| + |dy|상하좌우 이동만 가능
유클리드 거리sqrt(dx² + dy²)임의 방향의 직선 이동 비용을 근사
체비쇼프 거리max(|dx|, |dy|)직선과 대각선 한 칸 비용이 같음
옥타일 거리max(dx,dy)+(sqrt(2)-1)min(dx,dy)직선 1, 대각선 √2 비용

휴리스틱은 장애물을 정확히 통과하는 경로를 미리 찾는 함수가 아니라, 빠르고 안전한 하한 추정이어야 합니다. 모든 이동 비용에 공통 최솟값이 있다면 거리 공식에 그 비용을 곱해 실제 비용 단위와 맞춥니다.

최단 경로를 보장하는 조건

허용적 휴리스틱

휴리스틱이 실제 남은 최단 비용을 과대평가하지 않는 성질을 허용성(admissibility)이라고 합니다.

0 <= h(n) <= n에서 목적지까지의 실제 최단 비용

상하좌우 한 칸 비용이 1인 격자에서 맨해튼 거리는 장애물을 무시한 최소 이동 횟수입니다. 장애물 때문에 실제 경로가 더 길어질 수는 있어도 더 짧아지지는 않으므로 허용적입니다.

일관적 휴리스틱

인접한 두 정점 n, m에 대해 다음 부등식을 만족하면 일관적(consistent)이라고 합니다.

h(n) <= cost(n, m) + h(m)

한 번 이동한 뒤 예상 비용이 간선 비용보다 더 큰 폭으로 갑자기 감소하지 않는다는 뜻입니다. 일관성은 허용성보다 강한 조건이며, 그래프 탐색에서 큐에서 꺼낸 정점을 닫은 뒤 다시 열지 않고 처리하기 쉽게 만듭니다. 맨해튼·유클리드 거리는 각각 알맞은 이동 비용 조건에서 일관성을 만족합니다.

BFS·다익스트라·A* 비교

알고리즘우선 기준가중치 조건주요 용도
BFS시작점에서의 간선 수모든 간선 비용이 동일비가중 그래프의 최단 거리
다익스트라g(n)음수 간선 없음한 시작점에서 여러 목적지까지의 거리
A*g(n)+h(n)음수 간선 없음, 안전한 휴리스틱알려진 한 목적지까지의 경로 탐색

동일 비용 격자에서 목적지 하나만 찾는다면 BFS도 정답을 구하지만, 맵이 크고 좋은 휴리스틱이 있다면 A*가 훨씬 적은 칸을 방문할 수 있습니다. 여러 목적지의 거리를 한 번에 구해야 한다면 매번 A*를 실행하기보다 다익스트라 한 번이 더 적합할 수 있습니다.

우선순위 큐의 오래된 항목과 타이브레이크

같은 좌표에 더 짧은 경로가 발견될 때마다 새 상태를 큐에 넣으면 예전 g를 가진 항목도 남습니다. 큐에서 꺼낸 g가 현재 저장된 값과 다르면 오래된 정보이므로 건너뜁니다.

if (current.g != gScore[current.y][current.x]) {
    continue;
}

f가 같은 후보가 여럿이면 h가 작은 쪽, 즉 목적지에 가까운 쪽을 우선하거나 같은 의미로 g가 큰 쪽을 우선할 수 있습니다. 이는 탐색 모양과 방문 수에 영향을 주지만, 안전한 휴리스틱과 올바른 g 갱신이라는 최단 경로 조건은 바꾸지 않습니다.

상하좌우 격자용 C++ 구현

#include <algorithm>
#include <array>
#include <cstdlib>
#include <limits>
#include <queue>
#include <vector>

struct Point {
    int x;
    int y;
};

struct State {
    Point point;
    int g;
    int f;

    bool operator<(const State& other) const {
        if (f != other.f) {
            return f > other.f; // 작은 f가 먼저 나오는 최소 힙
        }
        return g < other.g;     // f가 같으면 큰 g를 우선
    }
};

int Manhattan(Point from, Point goal) {
    return std::abs(from.x - goal.x) + std::abs(from.y - goal.y);
}

std::vector<Point> FindPath(
    const std::vector<std::vector<int>>& grid,
    Point start,
    Point goal) {
    if (grid.empty() || grid.front().empty()) {
        return {};
    }

    const int height = static_cast<int>(grid.size());
    const int width = static_cast<int>(grid.front().size());
    const auto inBounds = [width, height](Point p) {
        return p.x >= 0 && p.x < width && p.y >= 0 && p.y < height;
    };

    if (!inBounds(start) || !inBounds(goal)
        || grid[start.y][start.x] != 0 || grid[goal.y][goal.x] != 0) {
        return {};
    }

    const int INF = std::numeric_limits<int>::max();
    std::vector<std::vector<int>> gScore(
        height, std::vector<int>(width, INF));
    std::vector<std::vector<Point>> parent(
        height, std::vector<Point>(width, {-1, -1}));
    std::priority_queue<State> open;

    gScore[start.y][start.x] = 0;
    open.push({start, 0, Manhattan(start, goal)});

    constexpr std::array<Point, 4> directions = {{
        {1, 0}, {-1, 0}, {0, 1}, {0, -1}
    }};

    while (!open.empty()) {
        const State current = open.top();
        open.pop();

        const Point p = current.point;
        if (current.g != gScore[p.y][p.x]) {
            continue;
        }
        if (p.x == goal.x && p.y == goal.y) {
            std::vector<Point> path;
            for (Point at = goal; at.x != -1; at = parent[at.y][at.x]) {
                path.push_back(at);
            }
            std::reverse(path.begin(), path.end());
            return path;
        }

        for (Point direction : directions) {
            const Point next = {p.x + direction.x, p.y + direction.y};
            if (!inBounds(next) || grid[next.y][next.x] != 0) {
                continue;
            }

            const int newG = current.g + 1;
            if (newG < gScore[next.y][next.x]) {
                gScore[next.y][next.x] = newG;
                parent[next.y][next.x] = p;
                open.push({next, newG, newG + Manhattan(next, goal)});
            }
        }
    }

    return {}; // 목적지에 도달할 수 없음
}

이 예제에서 0은 이동 가능, 다른 값은 장애물입니다. 직사각형 격자와 단위 이동 비용을 전제로 하며, 가중치가 다른 지형을 지원하려면 newG에 목적지 칸의 이동 비용을 더하고 휴리스틱에도 최소 이동 비용을 반영해야 합니다.

시간 복잡도와 실제 성능

인접 리스트와 우선순위 큐를 사용하는 일반 그래프 구현의 상한은 보통 O((V+E) log V)로 표현할 수 있고 공간은 O(V+E)입니다. 하지만 A*의 실질적인 장점은 점근 표기보다 목표에 도달하기 전에 몇 개의 정점을 확장하는가에 있습니다.

  • h=0: 다익스트라처럼 넓게 탐색한다.
  • 실제 비용에 가까운 안전한 h: 목적지 방향에 집중한다.
  • 정보가 약한 h: 최단 경로는 찾지만 불필요한 정점을 많이 방문한다.
  • 과대평가하는 h: 더 빨라질 수 있지만 최단 경로 보장이 깨질 수 있다.

장애물이 복잡하거나 휴리스틱으로 방향을 구분하기 어려운 맵에서는 최악의 경우 다익스트라에 가까운 범위를 탐색합니다. 휴리스틱 계산 자체도 매 정점에서 수행되므로 충분히 저렴해야 합니다.

자주 하는 실수

  • 이동 규칙과 휴리스틱이 맞지 않는다

    대각선 비용이 1인데 맨해튼 거리를 쓰면 실제 남은 비용을 과대평가할 수 있습니다.

  • f를 기준으로 경로를 갱신한다

    실제 최단 거리 비교는 반드시 newG < g[next]로 해야 합니다. f는 탐색 순서를 정하는 값입니다.

  • 목적지를 발견하자마자 종료한다

    이웃에서 목적지를 보았을 때가 아니라 목적지가 큐의 최우선 상태로 꺼내졌을 때 종료해야 합니다.

  • 오래된 큐 항목을 그대로 처리한다

    정답은 유지될 수 있어도 같은 정점의 이웃을 불필요하게 반복 탐색합니다.

  • 부모 정보를 갱신하지 않는다

    실제 이동 경로가 필요하다면 더 짧은 g를 찾을 때마다 parent도 함께 바꿉니다.

  • 휴리스틱에 장애물을 과하게 계산한다

    휴리스틱 자체가 최단 경로 탐색만큼 비싸면 A*로 탐색량을 줄이는 의미가 사라집니다.

정리

  • A*는 실제 누적 비용 g와 목적지까지의 추정 비용 h를 더한 f로 탐색 순서를 정한다.
  • h=0이면 다익스트라와 같고, 좋은 휴리스틱일수록 목적지 방향에 집중한다.
  • 최단 경로를 보장하려면 휴리스틱이 실제 남은 비용을 과대평가하지 않아야 한다.
  • 경로 갱신은 f가 아니라 실제 비용 g를 기준으로 판단한다.
  • 목적지가 큐에서 꺼내졌을 때 종료하고 parent를 역추적해 경로를 복원한다.

결국 A*는 다익스트라의 정확한 비용 계산에 문제 도메인의 방향 감각을 더한 알고리즘입니다. 휴리스틱을 공격적으로 만드는 것보다 이동 규칙에 맞는 안전한 하한을 고르는 것이 먼저입니다.
