Project Case Study

Roguelike Project

Unity 6000.3.12f1 LTS · C# · ScriptableObject · TextMeshPro

전투 중 얻는 카드, 상태이상, 유물 효과가 서로 반응하며 하나의 시너지 엔진으로 이어지는 턴제 로그라이크 프로젝트입니다. 단일 효과보다 잘 엮인 선택이 더 큰 보상으로 돌아오도록 전투 구조와 데이터 기반 확장성을 설계하고 있습니다.

프로젝트 개요

Roguelike Project는 플레이어가 전투 중 획득한 효과를 조합해 중후반에 강력한 시너지 빌드를 완성하는 것을 핵심 재미로 삼는 게임입니다. 핵심 설계 방향은 "좋은 단일 선택"보다 "잘 엮인 선택"을 보상하는 구조입니다.

  • 개별 효과는 단독으로 과도하게 강하지 않게 설계
  • 여러 효과가 연결되면 폭발력을 갖는 전투 흐름 구성
  • 상태이상, 패시브, 유물, 보상이 같은 시너지 방향을 만들도록 연결

목표와 역할

이 프로젝트의 목적은 Unity 포트폴리오에서 전투 시스템, 상태이상, 패시브, 이벤트 기반 구조를 직접 설계하고 구현할 수 있음을 보여주는 것입니다. 단순한 전투 루프 구현보다 확장 가능한 구조와 시스템 간 상호작용을 코드로 표현하는 데 집중했습니다.

  • 전투 루프와 전투 상태 관리 설계
  • 카드 효과, 상태이상, 유물 효과의 확장 구조 구현
  • ScriptableObject 기반 데이터 구조와 런타임 effect 조립 흐름 구성
  • 보상 선택과 빌드 방향성 선택 구조 구현

현재 구현

  • 전투 시스템

    턴 기반 전투 루프, 에너지 기반 행동, 플레이어/적 유닛 구조, PlayerTurn/EnemyTurn/Win/Lose 전투 상태를 구현했습니다.

  • 카드 시스템

    ScriptableObject 카드 데이터와 Effect 조합 구조를 사용해 조건, 동적 계산, 전용 효과를 카드별로 확장할 수 있게 했습니다.

  • 상태이상 시스템

    Poison, Burn, Vulnerable을 구현하고, 상태이상 간 시너지 로직을 전투 설계에 연결했습니다.

  • 유물과 보상

    IRelicEffect 기반 이벤트 트리거 구조, 유물 보상 선택, 중복 유물 방지, 전투 종료 후 카드 선택을 구현했습니다.

시스템 설계

카드와 유물은 ScriptableObject 데이터에서 런타임 effect를 조립하는 방식으로 통일했습니다. 전투 시스템은 구체 카드나 유물 종류를 직접 알기보다, 공통 인터페이스를 통해 효과 실행과 이벤트 반응을 처리합니다.

  • CardDataSO: 여러 CardEffectDataSO를 보유하고 런타임 ICardEffect를 생성
  • UnitStatusData: Unit의 상태이상 값을 Dictionary 구조로 관리
  • RelicDataSO: 여러 RelicEffectDataSO를 통해 이벤트 기반 IRelicEffect를 조립
  • BattleManager / StatusEffectController / RelicManager: 전투 흐름, 상태 처리, 유물 반응 책임 분리

핵심 설계 포인트는 데이터 기반 구조, 이벤트 기반 상호작용, 기존 코드 수정 최소화, 상태 스냅샷 기반 조건 처리입니다. 이 구조를 통해 카드 추가와 유물 효과 확장이 전투 루프 수정으로 번지지 않도록 했습니다.

개발 기록

  • 카드 효과 처리 구조를 switch 기반에서 Effect 조합 구조로 리팩터링했습니다.
  • Poison, Burn, Vulnerable 상태이상을 구현하고 상태이상 기반 전투 구조로 전환했습니다.
  • BattleManager, StatusEffectController, RelicManager의 책임을 분리해 결합도를 낮췄습니다.
  • 전투 종료 후 카드 선택, 유물 보상, 시작 패시브 선택으로 빌드 방향성 선택 구조를 만들었습니다.
  • 유물 시스템을 캐시 기반 구조에서 이벤트 기반 구조로 전환하고 카드 시스템과 구조를 맞췄습니다.

트러블슈팅

시너지 기반 전투는 효과가 서로를 다시 호출하기 쉬워서, 트리거 시점과 조건 검사 기준을 명확히 분리하는 것이 중요했습니다. 특히 유물과 상태이상 효과가 같은 이벤트에 반응할 때 중복 적용과 재귀 호출을 방지하는 처리가 필요했습니다.

  • OnApplyPoison 루프에서 발생한 유물 재귀 호출 문제 해결
  • 조건 검사 시점 오류를 상태 스냅샷 구조로 수정
  • PoisonCore / VenomSac 중복 적용 문제 해결
  • PressurePoint 턴 제한 버그 수정

다음 단계

MVP 기준으로 상태이상, 유물, 보상 흐름은 구현되어 있으며, 앞으로는 적 행동 패턴 다양화와 시너지 빌드 분기 구조를 확장할 예정입니다. 전투 로그 UI, 스킬 버튼, 현재 상태이상 표시 UI도 함께 보강해 플레이어가 시너지 흐름을 더 쉽게 읽을 수 있게 만들 계획입니다.
