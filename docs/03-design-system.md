# 디자인 시스템

## 방향

게임 개발자다운 분위기는 콘텐츠와 사례 이미지로 만들고, 인터페이스는 차분하고 읽기 쉽게 유지한다. 장식과 애니메이션보다 정보 위계와 대비를 우선한다.

## 토큰

색상, 최대 너비, 간격, 모서리 값은 `styles.css`의 `:root` 사용자 지정 속성을 기준으로 한다.

- 배경: 짙은 네이비 계열
- 본문: 밝은 회백색
- 보조 글자: 낮은 대비의 청회색
- 강조: 링크와 키보드 포커스에 사용하는 밝은 청록색
- 콘텐츠 최대 너비: `70rem`

## 타이포그래피

시스템 글꼴을 사용해 외부 폰트 요청을 없앤다. 본문 줄 높이는 충분히 확보하고 한 줄 길이는 대략 65~75자로 제한한다. 제목은 크기뿐 아니라 여백으로도 계층을 구분한다.

## 레이아웃과 컴포넌트

- 페이지 좌우 여백은 모바일에서 작게, 넓은 화면에서 점진적으로 늘린다.
- 소개 영역은 넓은 화면에서 텍스트와 프로필 사진의 2열, 좁은 화면에서 1열로 배치한다.
- 카드 목록은 좁은 화면에서 한 열, 공간이 생기면 자동으로 여러 열이 된다.
- 직무 및 기술 배지는 같은 테두리와 은은한 확대 하이라이트를 사용하며, 기술 목록은 화면 폭에 따라 줄바꿈한다.
- 링크는 색상만이 아니라 밑줄 또는 형태 변화로 구분한다.
- 모든 인터랙티브 요소에는 명확한 `:focus-visible` 상태를 둔다.

## 반응형 기준

기본 스타일을 모바일 기준으로 작성하고 `48rem` 이상에서 헤더와 소개 영역의 여백을 확장한다. 특정 기기보다 콘텐츠가 깨지는 지점을 기준으로 조정한다.
