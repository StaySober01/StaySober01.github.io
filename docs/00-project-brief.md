# 프로젝트 브리프

## 목적

게임 개발자로서 만든 프로젝트와 문제 해결 과정을 보여주는 개인 포트폴리오 블로그다. 방문자는 짧은 시간 안에 개발 분야, 대표 작업, 기술적 강점을 파악할 수 있어야 한다.

## 주요 독자

- 게임 스튜디오 채용 담당자와 개발자
- 협업 또는 프로젝트 참여를 검토하는 사람
- 기술 글과 TIL을 찾는 동료 개발자

## 핵심 콘텐츠

- 프로젝트 사례: 문제, 역할, 구현, 트러블슈팅, 결과, 배운 점
- TIL: 짧고 재사용 가능한 학습 기록
- 기술 블로그: 하나의 주제를 맥락과 근거를 갖춰 설명하는 글
- About: 관심 분야, 기술 역량, 경력 또는 학습 과정, 연락 경로

## 현재 범위

첫 단계는 `index.html`과 `styles.css`만 사용하는 정적 랜딩 페이지다. 실제 프로젝트 상세 페이지와 글 목록은 콘텐츠가 준비되는 순서대로 추가한다.

## 성공 기준

- 모바일과 데스크톱에서 핵심 섹션을 쉽게 탐색할 수 있다.
- 별도 빌드 없이 GitHub Pages에서 제공할 수 있다.
- 새 콘텐츠를 추가할 위치와 작성 기준이 문서에 정의되어 있다.
- 불필요한 스크립트와 외부 런타임 의존성이 없다.

## 제외 범위

현재 단계에서는 CMS, 댓글, 검색, 다국어, 복잡한 애니메이션, 프레임워크를 도입하지 않는다.
