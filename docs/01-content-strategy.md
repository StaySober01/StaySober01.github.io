# 콘텐츠 전략

## 원칙

- 결과 이미지보다 어떤 문제를 어떻게 해결했는지를 먼저 설명한다.
- 글 하나에는 명확한 독자와 핵심 메시지 하나를 둔다.
- 검증 가능한 수치, 코드, 영상 또는 회고를 가능한 범위에서 제시한다.
- 작성일과 수정일을 표시해 정보의 시점을 알 수 있게 한다.

## 콘텐츠 유형

### 프로젝트 사례

문제, 역할, 기간, 기술 스택, 핵심 구현, 트러블슈팅, 결과, 배운 점 순서를 기본 골격으로 사용한다. 팀 작업은 본인이 담당한 범위를 분명히 밝힌다.

### TIL

작은 문제 하나를 짧게 기록한다. 상황, 핵심 해법, 적용 시 주의점을 포함하고 관련 프로젝트나 긴 글로 연결한다.

### 기술 블로그

배경, 문제 정의, 후보 접근법, 선택 근거, 구현, 검증, 결론을 담는다. 코드가 있으면 실행 환경과 전제 조건을 함께 적는다.

### About

게임 개발 관심 분야, 잘하는 일, 사용 기술, 현재 목표, GitHub 및 연락 경로를 간결하게 제공한다.

## 발행 흐름

초안 작성 → 사실과 링크 확인 → 모바일 가독성 확인 → 발행 → 변경이 생기면 수정일 갱신의 흐름을 따른다. 초기에는 HTML로 관리하고, 글 수가 늘어 반복 작업이 커지면 Markdown 기반 전환을 검토한다.
