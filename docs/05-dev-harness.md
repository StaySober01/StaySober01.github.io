# 개발 하네스

## 작업 시작

1. `AGENTS.md`와 관련 문서를 읽는다.
2. `git status`와 현재 파일 구조를 확인한다.
3. 변경 목표와 검증 방법을 작은 단위로 정한다.

## 로컬 실행

빌드가 필요 없는 사이트이므로 `index.html`을 직접 열 수 있다. 링크와 실제 HTTP 제공 환경을 함께 확인하려면 Python이 설치된 환경에서 다음처럼 실행한다.

```bash
python -m http.server 8000
```

그 후 `http://localhost:8000/`에서 확인한다. Python은 프로젝트 의존성이 아니며 다른 정적 서버를 사용해도 된다.

## 완료 체크리스트

- 변경한 HTML의 문서 구조와 내부 링크를 확인한다.
- 좁은 화면과 넓은 화면에서 가로 스크롤, 겹침, 잘림이 없는지 확인한다.
- 키보드만으로 내비게이션과 링크에 접근할 수 있는지 확인한다.
- CSS, 이미지, 내부 링크가 상대 경로로 정상 로드되는지 확인한다.
- 코드와 관련 문서가 같은 상태를 설명하는지 확인한다.
- `git diff`로 요청 밖의 변경이 없는지 확인한다.

## 자동화 확장

현재는 `package.json`이 없어 `npm run build`, `lint`, `check` 명령도 없다. 프레임워크나 검사 도구를 도입할 때 명령, 요구 버전, CI 실행 방식을 이 문서에 추가한다.
